Generate a git commit message for the currently staged changes.

## Process

1. Run `git diff --cached` to see what's staged
2. If nothing is staged, run `git status` and ask the user what to stage
3. Analyze the changes and generate a commit message following Conventional Commits format:
   - `feat:` new feature
   - `fix:` bug fix
   - `refactor:` code restructure without behavior change
   - `test:` adding or updating tests
   - `docs:` documentation only
   - `chore:` tooling, deps, config
   - `perf:` performance improvement
   
4. For significant changes, use a body:
   ```
   feat(auth): add token refresh on 401 response
   
   Automatically retries failed requests with a refreshed token when
   the API returns a 401. Falls back to logout if refresh also fails.
   
   Closes #123
   ```

5. Show the proposed commit message and ask for approval or edits
6. Once approved, run `git commit -m "[message]"`

## Rules
- Subject line max 72 characters
- Use imperative mood ("add" not "added")  
- Reference issue numbers if a plan doc or progress doc mentions them
- Do not include "Generated by AI" or similar in the message
